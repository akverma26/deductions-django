GLSL

Shaders are pieces of program (using a C-like language) that are build onto the GPU and executed during the rendering pipeline. Depending on the nature of the shaders (there are many types depending on the version of OpenGL you're using), they will act at different stage of the rendering pipeline.

There are different types of Shaders like vertex and fragment.

 

In python how shaders are defined ?

xxxxxxxxxx
attribute vec2 position;
attribute vec4 color;
uniform float scale;
varying vec4 v_color;
​
void main() {
    gl_Position = vec4(position, 0, 1);
}

In above example, shader is defined for vertices.

Explanation:

Attribute: We need to tell the vertex shader that it will have to handle vertices where position is a tuple of 2 floats and color is a tuple of 4 floats. This is precisely what attributes are meant for.

xxxxxxxxxx
attribute vec2 position;
attribute vec4 color;

Uniform: Second type of information that can be fed. Let's say for example we want to scale all the vertices by a constant factor scale, we would thus write:

xxxxxxxxxx
uniform float scale;

Varying:

Last type is the varying type that is used to pass information between the vertex stage and the fragment stage. So let us suppose (again) we want to pass the vertex color to the fragment shader, we now write:

xxxxxxxxxx
varying vec4 v_color;

and now in the fragment_shader we write:

xxxxxxxxxx
varying vec4 v_color;
​
void main() {
    gl_FragColor = v_color;
}

Note: gl_Position and gl_FragColor are special type of variable in GLSL

Compiling Shader Program (withing python)

We wrote our shader and we need now to build a program that will link the vertex and the fragment shader together.

First we need to request program and shader slots from the GPU:

xxxxxxxxxx
program  = gl.glCreateProgram()
vertex   = gl.glCreateShader(gl.GL_VERTEX_SHADER)
fragment = gl.glCreateShader(gl.GL_FRAGMENT_SHADER)

We can now ask for the compilation of our shaders into GPU objects and we log for any error from the compiler (e.g. syntax error, undefined variables, etc):

xxxxxxxxxx
# Set shaders source
gl.glShaderSource(vertex, vertexShader)
gl.glShaderSource(fragment, fragmentShader)
​
# Compile shaders
gl.glCompileShader(vertex)
if not gl.glGetShaderiv(vertex, gl.GL_COMPILE_STATUS):
 error = gl.glGetShaderInfoLog(vertex).decode()
 print(error)
 raise RuntimeError("Vertex shader compilation error")
​
gl.glCompileShader(fragment)
if not gl.glGetShaderiv(fragment, gl.GL_COMPILE_STATUS):
 error = gl.glGetShaderInfoLog(fragment).decode()
 print(error)
 raise RuntimeError("Fragment shader compilation error")

Then we link our two objects in into a program and again, we check for errors during the process.

xxxxxxxxxx
gl.glAttachShader(program, vertex)
gl.glAttachShader(program, fragment)
gl.glLinkProgram(program)
​
if not gl.glGetProgramiv(program, gl.GL_LINK_STATUS):
 print(gl.glGetProgramInfoLog(program))
 raise RuntimeError('Linking error')

and we can get rid of the shaders, they won't be used again (you can think of them as .o files in C).

xxxxxxxxxx
gl.glDetachShader(program, vertex)
gl.glDetachShader(program, fragment)

Finally, we make program the default program to be ran. We can do it now because we'll use a single program in this example:

xxxxxxxxxx
gl.glUseProgram(program)

All the above things are done using one command using Glumpy library.

We still need to open a window, but now this is straightforward:

xxxxxxxxxx
from glumpy import app, gloo, gl
​
# Create a window with a valid GL context
window = app.Window()

Glumpy will look for any available backend in a given order.

If necessary, you can also indicate which backend to use by writing app.use("glfw") before creating the window.

xxxxxxxxxx
# Build the program and corresponding buffers (with 4 vertices)
quad = gloo.Program(vertexShader, fragmentShader, count=4)

With the above line, both the CPU data and GPU data (buffer) have been created and no extra command is necessary at this stage and uploading the data is only a matter of setting the different fields of the quad program:

xxxxxxxxxx
# Upload data into GPU
quad['position'] = (-1,+1), (+1,+1), (-1,-1), (+1,-1)

Under the hood, glumpy has parsed your shader programs and has identified attributes. Rendering is just a matter of calling the draw method from our shader program, using the proper mode.

xxxxxxxxxx
# Tell glumpy what needs to be done at each redraw
@window.event
def on_draw(dt):
    window.clear()
    quad.draw(gl.GL_TRIANGLE_STRIP)
​
# Run the app
app.run()

Now it is the time to run the program and the complete code is as following:

xxxxxxxxxx
from glumpy import app, gloo, gl
​
vertex = """
    attribute vec2 position;
    void main(){ gl_Position = vec4(position, 0.0, 1.0); } """
​
fragment = """
    void main() { gl_FragColor = vec4(1.0, 0.0, 0.0, 1.0); } """
​
# Create a window with a valid GL context
window = app.Window()
​
# Build the program and corresponding buffers (with 4 vertices)
quad = gloo.Program(vertex, fragment, count=4)
​
# Upload data into GPU
quad['position'] = (-1,+1), (+1,+1), (-1,-1), (+1,-1)
​
# Tell glumpy what needs to be done at each redraw
@window.event
def on_draw(dt):
    window.clear()
    quad.draw(gl.GL_TRIANGLE_STRIP)
​
# Run the app
app.run()
